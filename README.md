🕵️ Anonymous Post & Review Platform
A high-performance, privacy-focused backend designed to facilitate honest conversations without revealing user identities. This MVP was built with a focus on security-first architecture and database optimization.

🚀 The Purpose
The platform provides a secure digital space where users can post content and share reviews anonymously. By prioritizing a "Backend-First" development approach, the app ensures sub-50ms data retrieval, robust encryption, and strict data integrity.

✨ Features
True Anonymity: Public-facing content hides author IDs while maintaining a secure link in the database.

Peppered Authentication: Advanced security using bcrypt combined with a server-side SECRET_SALT (Pepper) to prevent rainbow table attacks.

Optimized Commenting: Dedicated routes for fetching post-specific comments with high-speed indexing.

Role-Based Access Control (RBAC): Distinct permissions for STUDENT and ADMIN roles.

Cascading Deletes: Automated database cleanup; deleting a post removes all associated anonymous reviews instantly.

🛠 Tech Stack
Runtime: Node.js (Express.js)

ORM: Prisma 6 (Latest)

Database: PostgreSQL

Security: JSON Web Tokens (JWT) & Bcrypt

Environment: Secure .env management for Pepper and Secret Keys

🔒 Security Implementation
To ensure maximum protection, this project implements a Two-Factor Hashing strategy:

Salt: Automatically generated by Bcrypt (stored in DB).

Pepper: A secret environment variable (SECRET_SALT) added to the password string before hashing.

JWT: Signed with a JWT_SECRET to ensure token payloads cannot be manipulated by users.

📈 Optimization
Indexing: @@index([postId]) and @@index([authorId]) are used to ensure constant-time lookups even as the dataset grows.

Selective Fetching: The API uses Prisma's .select() to return only necessary fields, reducing payload size and increasing response speed.
